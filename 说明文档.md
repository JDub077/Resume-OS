# 履历星球 Resume OS

## AI 原生大学生成长经历资产管理平台

> **让每一段经历都被记录，让每一次申请都无需重写。**

**线上体验**：https://jdub077.github.io/Resume-OS/

---

## 一、项目概述

履历星球 Resume OS 是一款面向大学生的 **AI 原生成长经历资产管理平台**。它解决的不仅是一个"写材料"的效率问题，而是大学生在评奖评优、求职升学过程中反复面对的底层痛点：**同一段经历被无数次重写、经历碎片化散落各处、有经历却不会专业表达**。

产品以 **"经历沉淀 → AI 智能表达 → 持续成长"** 为核心闭环，让大学生的每一段成长都被结构化记录，并在需要时通过 AI 自动包装为不同场景的高质量申请材料。

## 二、用户需求与痛点分析

### 2.1 目标用户

- **评奖评优型**：频繁申请国家奖学金、优秀学生干部、优秀志愿者等
- **求职就业型**：需要反复投递简历、填写网申、准备面试自我介绍
- **升学深造型**：撰写保研个人陈述、考研复试材料、留学申请 CV

### 2.2 四大核心痛点

| 痛点 | 具体表现 | 后果 |
|------|---------|------|
| **重复劳动严重** | 同一段学生会部长经历，需要分别写入奖学金申请、干部申报、求职简历、面试介绍 | 大学四年平均重写同一段经历 30+ 次，浪费大量时间 |
| **经历碎片化散落** | 经历分散在 Word 文档、微信聊天记录、手机备忘录、旧申请材料中 | 需要时找不到、记不全、漏掉关键成果数据 |
| **不会专业表达** | "负责招新" vs "主导学院社团招新活动策划与执行，覆盖新生 300+ 人次" | 优秀经历被平淡表达埋没，竞争力大打折扣 |
| **材料无法动态升级** | 去年写过的材料，今年新增了实习/竞赛经历后，仍需全部重写 | 无法复用已有材料，每次从零开始 |

### 2.3 场景还原

> **小明的困扰**：大三下学期，小明同时面临三件事——国家奖学金申请截止、腾讯暑期实习网申开放、保研夏令营材料提交。这三份材料都需要写入他的学生会外联部部长经历和腾讯产品运营实习经历。但他发现：奖学金材料在夸克网盘里，简历在 WPS 里，保研陈述在备忘录里，三段经历的描述口径完全不一致，成果数据也记混了。他不得不花了整整两个通宵，把同两段经历重写了三遍。

这就是履历星球要解决的问题。

## 三、产品解决方案

### 3.1 核心定位

面向大学生的 **AI 经历资产管理平台**，核心逻辑：

```
碎片化经历（文本/文件）
    ↓ AI 结构化解析
经历资产库（分类管理、标签检索）
    ↓ AI 场景化生成
奖学金申请 / 求职简历 / 保研陈述 / 面试自我介绍...
    ↓ 持续沉淀
经历增加 → 材料自动升级
```

### 3.2 三步走价值主张

**第一步：一次录入，永久沉淀**
- 支持自然语言文本输入，AI 自动解析为结构化字段
- 经历资产库统一管理，分类清晰、检索便捷

**第二步：AI 智能表达，千人千面**
- 同一批经历，根据目标场景（奖学金/求职/保研）自动调整表达风格
- 根据目标岗位方向（如"前端开发岗"vs"产品经理岗"）智能推荐和包装不同经历

**第三步：持续成长，自动升级**
- 新增经历后，历史材料可一键刷新，无需重写
- AI 成长诊断雷达图，主动发现能力缺口并给出成长建议

## 五、AI 能力设计（AI 原生性核心体现）

产品不是"AI + 工具"，而是**以 AI 为核心能力原生搭建**的平台。AI 贯穿用户使用全链路：

### 5.1 AI 经历解析

用户粘贴自然语言文本，AI 自动提取 6 个结构化字段：
- **title**：提炼核心事件名称（8-15 字），避免简单截取原文
- **category**：判断经历类型（学生工作/志愿服务/实习经历/项目实践/获奖荣誉/自定义）
- **time**：识别时间并格式化为 "YYYY.MM"
- **description**：1-2 句话概括核心职责与动作
- **tags**：提炼 3-5 个精准能力标签（如"前端开发"而非"编程"）
- **result**：提取量化成果（数字、百分比、排名）

**Prompt 工程**：采用 Few-shot 示例 + 严格字段约束 + temperature=0.2，确保输出稳定可预测。返回结果经过字段校验，缺失项自动 fallback 到本地规则补全。

### 5.2 AI 材料生成

**场景化 Prompt 模板**：8 种场景各配备专属 prompt 配置，控制语言风格、篇幅、重点：

| 场景 | 语言风格 | 重点突出 | 篇幅 |
|------|---------|---------|------|
| 国家奖学金 | 正式、严谨、有感染力 | 学术能力、综合素质、领导力 | 600 字以内 |
| 求职简历 | 专业、简洁、有竞争力 | 岗位匹配度、STAR 法则、成果量化 | 每段 50-100 字 |
| 保研陈述 | 学术化、严谨、有深度 | 学术潜力、科研兴趣、研究潜力 | 800 字以内 |
| 面试自我介绍 | 自信、自然、有亲和力 | 个人亮点、岗位匹配性 | 2 分钟 / 400 字 |

**目标方向增强**：用户在生成时可输入具体目标（如"前端开发岗"、"计算机视觉方向"），AI 会针对性选择和包装最相关的经历，实现"千人千面"的定制化生成。

### 5.3 AI 经历推荐排序

根据用户选择的用途，系统自动计算每条经历的相关性分数：
- **类别权重**：求职简历场景下，"实习经历"权重最高（5 分），"项目实践"次之（4 分）
- **标签匹配**：经历标签与场景高频能力词匹配时额外加分
- **一键推荐**：自动勾选相关性最高的 1-3 条经历，并标注"推荐"徽章

### 5.4 AI 润色表达

对经历描述进行专业润色，将平淡表达升级为申请材料级别的正式表达，保持原意不变。

### 5.5 AI 成长诊断

基于用户的经历分布数据，AI 自动分析能力缺口：
- 雷达图可视化 5 维分布
- 综合成长指数评分（0-100）
- 智能建议："实习经历为空，建议大三暑期补充对口实习"
- 诊断逻辑完全本地实现，无需 API 调用，即时响应

## 六、设计思路与交互流程

### 6.1 设计原则

- **极简交互**：核心流程控制在 4 步以内（选场景 → 选经历 → 生成 → 复制/保存）
- **即时反馈**：每次操作都有动画过渡和 Toast 提示，消除等待焦虑
- **渐进引导**：首次访问自动写入 4 条示例经历，降低上手门槛
- **移动端友好**：侧边栏自适应为底部导航，核心功能在手机上可用

### 6.2 核心页面交互

**Dashboard 首页**
- 顶部欢迎语 + 经历总量
- 分类数据看板（点击可筛选经历库）
- AI 成长诊断（雷达图 + 评分 + 建议）
- 成长时间线（按时间轴展示经历）
- 快捷操作入口

**经历录入页**
- 大文本输入框 + AI 解析按钮
- 解析中逐字段动画（标题 → 类型 → 时间 → 标签 → 成果 → 描述，每步 450ms）
- 解析结果表单可直接编辑调整
- AI 润色按钮一键升级表达

**经历库页**
- 顶部分类可视化数据栏（图标 + 数量 + 名称，点击筛选）
- 搜索框实时过滤
- 分类 Tab 栏辅助筛选
- 经历卡片展示标题、类型、时间、标签、成果
- 操作：编辑 / 删除 / 用于生成

**材料生成页**
- 4 步流程指示器
- 步骤1：选择用途 + 输入目标方向（选填但推荐）
- 步骤2：经历列表按相关性智能排序，高分经历标"推荐"，支持一键勾选
- 步骤3：生成中 Loading 动画 + 耗时显示
- 步骤4：结果展示，支持编辑、保存、复制，标注真实 AI / 演示模式

## 七、使用场景与产品价值

### 7.1 场景一：奖学金申请季

> **用户**：大二学生，要申请国家奖学金，需要写 600 字个人事迹材料。
> 
> **使用前**：翻遍微信记录和旧文档，找到 3 段经历，花了 2 小时拼凑材料，表达平淡。
> 
> **使用后**：打开履历星球 → 选择"国家奖学金" → 勾选已有经历 → AI 30 秒生成正式材料 → 微调后提交。全程 5 分钟。

### 7.2 场景二：秋招求职

> **用户**：大四毕业生，要投 20 家企业，每家企业网申都要写不同的个人陈述。
> 
> **使用前**：同一段实习经历重写了 20 遍，投到第 10 家时已经精疲力尽，开始敷衍。
> 
> **使用后**：录入一次实习经历 → 选择"求职简历"场景，输入目标方向"产品经理岗" → AI 自动包装为产品视角的 STAR 描述 → 保存到历史 → 换家企业时只改公司名即可复用。

### 7.3 场景三：保研材料准备

> **用户**：大三学生，准备申请保研夏令营，需要写 800 字个人陈述。
> 
> **使用前**：不知道学术材料怎么写，网上找模板又担心雷同。
> 
> **使用后**：选择"保研陈述"场景，输入方向"计算机视觉" → AI 自动突出科研项目经历，弱化社团活动，生成学术化表达的陈述稿。

### 7.4 场景四：成长规划

> **用户**：大一新生，刚入学，想规划大学四年如何充实简历。
> 
> **使用后**：Dashboard AI 诊断显示"实习经历 0、项目实践 0" → 根据建议，大二暑假去实习，大三参加学科竞赛 → 经历库持续充实，毕业时一键生成完整求职简历。

## 八、与 PCG 业务场景的结合

题目要求作品基于真实校园场景并与 PCG 业务场景结合。履历星球与 PCG 业务的结合体现在两个层面：

### 8.1 内容创作层：AI 驱动的 UGC 内容生产

PCG 的核心资产是内容。履历星球帮助大学生将个人成长经历转化为**高质量的 UGC 内容**：

- **分享卡片生成**：一键生成精美的成长履历图片，可直接发布到 **QQ 空间、朋友圈**（PCG 社交场景）
- **短视频脚本生成**：将经历转化为 1 分钟自我介绍短视频的分镜脚本（结合 **腾讯视频 / 微视** 的 AI 短剧内容生态）
- **图文内容沉淀**：结构化经历库可作为个人 IP 内容的素材库，持续输出成长类图文/播客（结合 **腾讯新闻、QQ 音乐播客**）

### 8.2 社交互动层：经历分享与互助点评

PCG 的另一核心资产是社交关系。履历星球设计了**经历分享的社交闭环**：

- 生成的材料/分享卡片可以下载后发布到社交平台
- 模板社区可扩展为"互相点评"模式，同学之间可以互相优化材料（结合 **QQ 群、QQ 频道** 的社群互动）
- 未来可接入 **QQ 小程序**，直接在 QQ 生态内完成经历录入和材料生成

### 8.3 业务落地潜力

- **To C 直接服务**：面向大学生提供简历/材料生成付费服务
- **To B 高校合作**：与高校就业中心、学工系统集成，作为官方推荐工具
- **To B 企业合作**：与校园招聘平台（如腾讯校招）对接，提供简历优化服务

| 数据存储 | localStorage | 比赛版轻量方案，可扩展至后端数据库 |
| AI 接口 | OpenAI 兼容格式 | 支持 SiliconFlow / Kimi / DeepSeek / OpenAI 等多厂商 |

| 部署托管 | GitHub Pages + GitHub Actions | 自动构建部署，免费稳定 |

### 9.2 代理服务模式（评委体验关键设计）

为避免评委需要自行申请 API Key，项目设计了 **Cloudflare Worker 代理模式**：
- API Key 由开发者配置在 Worker 环境变量中（Secret 类型），不暴露到前端代码
- 评委打开网页即可直接使用真实 AI 功能，零配置门槛
- 同时支持用户自行填写 Key 的直连模式，保护用户隐私选择

项目已完成全部功能开发和部署：
1. 前端 6 个页面完整实现
2. AI 解析/生成/润色/推荐/诊断全流程打通

## 十二、团队感言

> 大学生不是没有经历，而是不擅长表达经历。履历星球，让每一段成长都被看见。
