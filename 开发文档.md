# 履历星球 Resume OS — 开发计划

## Context

从零构建一个面向大学生的 AI 经历资产管理平台 MVP，解决"同一段经历反复改写几十次"的校园痛点。比赛需要提交可访问的 Demo 链接、3 分钟录屏和 PDF 说明文档。项目当前为空目录，需一次性完成代码开发并部署上线。

## 推荐方案

采用 React + Vite + TailwindCSS + shadcn/ui + Framer Motion + React Router 构建单页应用，数据存 localStorage，AI 调用兼容 OpenAI API 格式。

## Phase 1: 项目脚手架（1h）

1. 用 `npm create vite@latest resume-os -- --template react` 初始化
2. 安装依赖：`tailwindcss postcss autoprefixer framer-motion react-router-dom lucide-react`
3. 初始化 shadcn/ui（提供 Button/Card/Input/Textarea/Select/Dialog/Toast 等组件）
4. 配置 `vite.config.js` 和 `tailwind.config.js`
5. 创建目录结构：

```
src/
  components/ui/     # shadcn 组件
  components/        # 业务组件
  pages/             # 页面级组件
  hooks/             # 自定义 hooks
  utils/             # 工具函数
  data/              # mock 数据 & localStorage 封装
  App.jsx
  main.jsx
```

## Phase 2: 核心基础设施（1.5h）

1. **路由配置** — `App.jsx` 中配置 `/dashboard`, `/input`, `/library`, `/generate`, `/templates` 五个路由
2. **localStorage 封装** — `src/data/storage.js` 提供 CRUD API，经历数组以 `resume_os_experiences` 为 key 存储
3. **AI 调用封装** — `src/utils/ai.js` 提供 `callAI(prompt)` 函数，读取用户填写的 `apiKey` 和 `baseURL`，兼容 OpenAI SDK 格式，使用 `fetch` 实现（减少 bundle 体积）
4. **布局组件** — `Sidebar` 或 `BottomNav` 导航 + `Header` 用户欢迎语，使用 Framer Motion 做页面切换动画
5. **全局样式** — 定义主题色（主色深蓝 `#2563eb`，辅色紫 `#7c3aed`，背景灰 `#f8fafc`）

## Phase 3: 页面开发（4h）

### 页面1: Dashboard (`/dashboard`)
- 顶部欢迎语 + 累计经历数
- 4 个数据卡片（学生工作 / 志愿服务 / 项目实践 / 实习经历）
- 底部 3 个快捷按钮（添加经历 / 生成材料 / 浏览模板）
- 空状态引导（首次进入展示示例数据）

### 页面2: 经历录入 (`/input`)
- 大文本输入框（placeholder + 示例文字）
- "AI 解析经历" 按钮，点击后调用 AI 接口解析
- 解析结果展示为可编辑表单（标题、类型、时间、描述、标签、成果）
- 支持手动调整后再保存
- 文件上传按钮（UI 占位，前端假解析，直接将文件名作为标题）

### 页面3: 经历库 (`/library`)
- 顶部分类筛选 Tab（全部 / 学生工作 / 志愿服务 / 实习经历 / 项目实践 / 获奖荣誉）
- 经历卡片网格展示（标题、类型、时间、标签）
- 卡片操作：编辑（跳转录入页并回填）、删除（Dialog 确认）、生成（跳转生成页并预选中）
- 空状态插图

### 页面4: 材料生成 (`/generate`)
- 第一步：选择用途（单选：国家奖学金 / 优秀学生干部 / 求职简历 / 网申登记表 / 面试自我介绍 / 保研陈述）
- 第二步：选择经历（Checkbox 列表，支持从经历库预选中）
- 第三步：AI 生成（调用 Prompt 模板，显示 loading 动画）
- 第四步：结果展示（可复制的长文本 + "重新生成" 按钮）
- 提供默认 Prompt 模板，确保输出质量

### 页面5: 模板社区 (`/templates`)
- 模板卡片列表（标题、适用场景、热度标识）
- 点击模板后跳转生成页并自动选择对应场景
- 初始内置 6-8 个 mock 模板数据

## Phase 4: 数据与体验优化（1.5h）

1. **mock 数据** — 预置 3-4 条高质量示例经历，首次打开自动写入 localStorage
2. **Toast 通知** — 保存/删除/生成成功提示
3. **Loading 状态** — AI 调用期间显示"AI 正在思考..."动画
4. **响应式适配** — 确保 PC 端演示效果最佳，移动端可用
5. **API Key 配置** — 在 Header 或设置弹窗中让用户填写自己的 DeepSeek/OpenAI Key，比赛演示时提前填好

## Phase 5: 构建与部署（1h）

1. `npm run build` 验证无错误
2. 部署到 Vercel（推荐）或 Netlify / GitHub Pages
3. 验证线上链接可正常访问
4. 准备录屏环境（确保网络稳定、界面整洁）

## 关键文件路径

| 文件 | 说明 |
|------|------|
| `src/App.jsx` | 路由入口 |
| `src/data/storage.js` | localStorage CRUD |
| `src/utils/ai.js` | AI 调用封装 |
| `src/pages/Dashboard.jsx` | 首页 |
| `src/pages/ExperienceInput.jsx` | 经历录入 |
| `src/pages/ExperienceLibrary.jsx` | 经历库 |
| `src/pages/MaterialGenerate.jsx` | 材料生成 |
| `src/pages/TemplateCommunity.jsx` | 模板社区 |
| `src/components/Layout.jsx` | 整体布局 + 导航 |

## 验证方式

1. 本地 `npm run dev` 启动，检查 5 个页面路由是否正常
2. 添加一条经历，确认 Dashboard 统计更新、经历库显示正确
3. 选择经历和场景生成材料，确认 AI 返回结果展示正常
4. 刷新页面，确认数据通过 localStorage 持久化
5. 执行 `npm run build` 无报错，部署后线上验证
